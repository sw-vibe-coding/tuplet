Implemented the first parser-owned syntax registry. The parser now calls `Registry.add_syntax` for both `syntax ... expand ...` and `syntax ... -> ...` declarations, preserving declaration order, mode, template token slice, and expansion token slice. Parser smoke drivers use split-module qualification through the upstream multi-file runner instead of source concatenation. Added registry dump drivers and reg-rs baselines for expansion and verb registrations, and updated existing parser regressions to include the registry module.
